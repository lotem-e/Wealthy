## רקע וסטטוס נוכחי

האפליקציה קיימת ועובדת כקובץ HTML יחיד (`index.html`) — React 18 ו-Babel בצד לקוח, ללא build step. היא רצה כ-PWA (ניתן להוסיף למסך הבית ב-iPhone) ומשתמשת ב-Supabase לאימות ואחסון ענן, עם localStorage כ-cache מקומי.

### סטאק טכני
- **Frontend**: React 18 (UMD) + Babel standalone — קובץ HTML אחד, ללא Node/bundler
- **Auth**: Supabase Auth — Google OAuth
- **DB**: Supabase (PostgreSQL) — טבלת `app_data` עם שורה אחת לכל משתמשת, כל הנתונים כ-JSON בעמודת `data`
- **Hosting**: GitHub Pages (עתידי) / כל static host
- **פונט**: Heebo מ-Google Fonts

## פיצ'רים קיימים (מומשו)

### אימות
- Google OAuth דרך Supabase
- מסך התחברות עם כפתור "המשך עם Google"
- אפשרות "המשך בלי חשבון" (localStorage בלבד)
- סנכרון אוטומטי: בכניסה מושכים מהענן, בכל שינוי דוחפים (debounce 800ms)

### מבנה ניווט (עודכן יולי 2026)
- **ניווט תחתון** בסגנון iOS: קפסולת זכוכית (glass) עם 4 טאבים — מאזנים, תנועות, תקציבים, הגדרות — ולידה כפתור פלוס עגול נפרד שפותח את גיליון ההוספה
- אייקוני SVG בקו דק בסגנון SF Symbols (מצוירים ידנית — הפונט של אפל לא מורשה לווב)
- כותרת גדולה מתחלפת לפי הטאב הפעיל (Large Title)

### מאזנים (טאב הבית)
- **מאזן כולל** — סכום יתרות כל החשבונות + פירוט לכל חשבון
- זה הטאב היחיד שנשאר ממסך הבית הישן; כרטיס התקציב הגמיש והפילוחים הוסרו (הפילוחים עברו לטאב תנועות, התקציב ייבנה מחדש בטאב תקציבים)

### תנועות
- ניווט חודשי (קדימה/אחורה + כפתור "היום") — עבר לכאן מהכותרת הראשית
- פילטר לפי חשבון
- **ווידג'טים בקרוסלה אופקית** מעל הרשימה, מחושבים על הרשימה המסוננת בלבד:
  1. **סה"כ בתצוגה** — נטו (הכנסות פחות הוצאות, ללא העברות) + פירוט נכנס/יצא + מספר תנועות
  2. **פילוח לפי קטגוריות** — top 4 עם פסי התקדמות
  3. **פילוח לפי בתי עסק** — top 4
- רשימת תנועות של החודש הנצפה
- מחיקת תנועה (עם הוספה ל-fixedSkips אם היא קבועה)
- תגיות: תשלום X/Y, קבועה
- צבע שונה להכנסות (ירוק), העברות (אפור), הוצאות (כהה)

### תקציבים
- טאב ריק בינתיים — placeholder "בקרוב". כאן ייבנה פיצ'ר התקציבים המחודש

### הוספת תנועה (bottom sheet)
- **סוגים**: הוצאה / הכנסה / העברה
- שדות: סכום, חשבון, בית עסק (עם autocomplete מבתי עסק קיימים), קטגוריה, תאריך, הערה
- **תשלומים**: הוצאה ב-N תשלומים מתפרסת אוטומטית על N חודשים קדימה (תזרימי)
- קטגוריה אוטומטית אם בית העסק מוכר במערכת
- העברה: מחשבון ← לחשבון (לא נספרת כהוצאה)

### הדבקת הודעות SMS (📩)
- נפתחת מכפתור "הדבקת הודעות אשראי" בתוך גיליון הוספת התנועה (לא מהכותרת)
- textarea להדבקת הודעה אחת או כמה
- מפענח regex מקומי לפורמט **ישראכרט** בלבד (כרגע):
  ```
  בכרטיסך המסתיים ב- XXXX, אושרה עסקה ב-DD/MM בסך X.XX ש"ח בשם עסק.
  ```
- מזהה: סכום, בית עסק, תאריך, 4 ספרות כרטיס
- שיוך אוטומטי לחשבון לפי שדה "זיהוי" בהגדרות חשבון
- מסך אישור לפני שמירה (ניתן לערוך/למחוק שורות)
- **לא** שולח לענן — פענוח 100% מקומי

### הגדרות (לשעבר "ניהול")
- **תקציב חודשי כולל** — שדה עריכה
- **חשבונות** — עריכת שם, יתרת פתיחה, שדה זיהוי (4 ספרות לשיוך SMS); הוספת חשבון
- **הוצאות קבועות** — רשימה + הוספה (שם, סכום, קטגוריה, חשבון, יום בחודש); נוצרות אוטומטית בכל חודש ביום שנקבע; מחיקת תנועה קבועה מוסיפה אותה ל-fixedSkips
- **בתי עסק** — רשימה של בתי עסק שנצברו, עריכת קטגוריה לכל אחד
- **סנכרון ענן** — מייל המשתמשת, סטטוס (שומרת / מסונכרן ☁️✓ / לא מסונכרן), התנתקות
- **גיבוי ושחזור** — ייצוא JSON + ייבוא מקובץ

### PWA
- `apple-mobile-web-app-capable` — תמיכה ב-"Add to Home Screen" ב-iPhone
- `safe-area-inset` לניווט iPhone (notch/home bar)
- `theme-color` ירוק כהה
- עובד אופליין (localStorage כ-fallback)

## הוראות Deploy

### GitHub Pages
1. יוצרים ריפו ב-GitHub (יכול להיות Public)
2. מעלים את `index.html` לתיקיית root
3. Settings ← Pages ← Branch: main ← Save
4. האתר זמין בכתובת `https://<username>.github.io/<repo>/`
5. מעדכנים ב-Supabase: Authentication ← URL Configuration ← Site URL ← הכתובת החדשה

### Google OAuth — הערות
- ב-Google Console, OAuth App במצב "Testing" — צריך להוסיף מייל כ-Test User לפני שההתחברות עובדת

- Authorized redirect URI חייב להיות: `https://cgwaxjtebgvvtcdvfzvk.supabase.co/auth/v1/callback`
